# Contributing to CryptoGraphy

Thank you for your interest in contributing! Here is how you can help.

## 🐛 Reporting Bugs

Open an issue on the [Issues tab](../../issues) and include:

- Your operating system and version (e.g., Windows 11, Ubuntu 24.04)
- Steps to reproduce the bug
- What you expected vs. what actually happened
- Any error message shown in the app's log panel or in the terminal
- The file type you were processing (e.g., `.pdf`, `.jpg`) — no need to share the actual file

## 💡 Suggesting Features

Open an issue describing:
- What the new feature does
- Why it would be useful to users
- A rough idea of how it would work in the UI

## 🔧 Pull Requests

1. Click **Fork** at the top of the repository page
2. Make your changes in your fork
3. Test that encrypt → decrypt roundtrip works correctly for all algorithms
4. Open a **Pull Request** back to this repository
5. Describe your changes clearly

### Code Style
- Keep the single-file structure for small changes
- Use type hints and docstrings consistent with the existing code
- Test with both Fernet and AES-256-GCM, and with all name/extension modes

## 🌍 Translations

If you can translate the README into Persian (Farsi), Arabic, Russian, Chinese, or any other language, please contribute a file named `README.fa.md`, `README.ar.md`, etc.
